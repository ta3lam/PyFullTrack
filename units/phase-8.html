🚀 Comments & Code Style

Unit 1 — Set up your environment and write your first Python code. No experience needed!

Why Style Matters

Clean code is easier to read, debug, and share. Python has an official style guide called PEP 8.

Comments

python
# Single line comment — explain WHY not WHAT """ Multi-line docstring Used for functions, classes, and modules """ def calculate_profit(revenue, cost): """ Calculate profit margin as a percentage. Args: revenue (float): Total revenue cost (float): Total cost Returns: float: Profit margin percentage """ return (revenue - cost) / revenue * 100 # ✅ Good comment — explains WHY salary *= 1.15 # 15% raise approved in Q3 review # ❌ Bad comment — states the obvious salary *= 1.15 # multiply salary by 1.15

PEP 8 Key Rules

python
# ✅ Naming conventions my_variable = 10 # snake_case for variables MY_CONSTANT = 3.14 # UPPER_CASE for constants def my_function(): pass # snake_case for functions class MyClass: pass # PascalCase for classes # ✅ Spacing x = 1 + 2 # spaces around operators def func(x, y): # spaces after commas pass # ✅ Line length — max 79 characters # ✅ Two blank lines between top-level functions # ✅ One blank line between methods in a class
💡 Tip: Install autopep8 or black to auto-format your code: pip install black then black myfile.py

Readable Code Example

python
# ❌ Hard to read def calc(r,c,t=0.14): p=(r-c)/r*100;v=r*t;return p,v # ✅ Easy to read def calculate_financials(revenue, cost, vat_rate=0.14): """Calculate profit margin and VAT amount.""" profit_margin = (revenue - cost) / revenue * 100 vat_amount = revenue * vat_rate return profit_margin, vat_amount

Args: unit_price (float): Price per unit quantity (int): Number of units vat_rate (float): VAT rate (default 14%) Returns: tuple: (subtotal, vat_amount, total) """ subtotal = unit_price * quantity vat_amount = subtotal * vat_rate total = subtotal + vat_amount return subtotal, vat_amount, total # Calculate invoice for 5 units at 1000 EGP each unit_price = 1000 quantity = 5 subtotal, vat, total = calculate_invoice(unit_price, quantity) print(f"Subtotal: {subtotal:,}") print(f"VAT: {vat:,.2f}") print(f"Total: {total:,.2f}")

🚀 Virtual Environments & pip

Unit 1 — Set up your environment and write your first Python code. No experience needed!

What is a Virtual Environment?

A virtual environment is an isolated Python workspace. Each project gets its own packages — no conflicts between projects.

📝 Note: Think of it like separate toolboxes: Project A uses library v1.0, Project B uses v2.0 — they don't interfere.

Creating a Virtual Environment

python
# In Terminal / Anaconda Prompt: # Create environment named "myproject" conda create -n myproject python=3.11 # Activate it conda activate myproject # Windows/Mac/Linux # Deactivate when done conda deactivate

Installing Packages with pip

python
# Install a package pip install pandas # Install specific version pip install pandas==2.1.0 # Install multiple packages at once pip install pandas numpy matplotlib # Upgrade a package pip install --upgrade pandas # Uninstall pip uninstall pandas # See all installed packages pip list # Save project requirements to a file pip freeze > requirements.txt # Install from requirements file (share with team) pip install -r requirements.txt
💡 Tip: Always activate your environment before installing packages. Use conda activate env_name first.

Conda vs pip

python
# conda — better for data science (handles non-Python deps) conda install scikit-learn # pip — universal Python package manager pip install some-package # Check environment info conda info conda list

📊 Building Analysis Reports

Unit 7 — Extract real insights from data — grouping, pivoting, merging, time series, and statistical analysis.

Structuring an Analysis Report

python
import pandas as pd, numpy as np np.random.seed(42) def sales_report(df, title="Sales Report"): """Generate a complete sales analysis report.""" sep = "="*50 print(f"\n{sep}") print(f" {title}") print(sep) # 1. Overview print(f"\n📊 OVERVIEW") print(f" Period: {df.index.min().date()} to {df.index.max().date()}") print(f" Total: {df['Sales'].sum():>15,.0f}") print(f" Daily Avg: {df['Sales'].mean():>15,.0f}") print(f" Best Day: {df['Sales'].max():>15,.0f} ({df['Sales'].idxmax().date()})") # 2. Trend monthly = df["Sales"].resample("ME").sum() growth = monthly.pct_change().mean() * 100 print(f"\n📈 TREND") print(f" Monthly growth (avg): {growth:+.1f}%") # 3. Regional breakdown if "Region" in df.columns: print(f"\n🗺️ BY REGION") by_region = df.groupby("Region")["Sales"].agg(["sum","mean"]).round(0) by_region["Share%"] = (by_region["sum"]/by_region["sum"].sum()*100).round(1) print(by_region.sort_values("sum",ascending=False).to_string()) print(f"\n{sep}\n") return monthly
💡 Tip: A good report has: Overview → Trend → Breakdown → Insights → Recommendations. Always answer the 'so what' question.

🗄️ SQL with Python

Query databases directly from Python.

Why SQL + Python?

Most real-world data lives in databases. Python's sqlite3 module lets you query SQLite databases, and pandas can read SQL queries directly.

python
import sqlite3 import pandas as pd # Create in-memory database conn = sqlite3.connect(':memory:') # Create a table and insert data conn.executescript(""" CREATE TABLE sales ( id INTEGER PRIMARY KEY, product TEXT, amount REAL, month INTEGER, region TEXT ); INSERT INTO sales VALUES (1,'Laptop',1200,1,'Cairo'), (2,'Phone',800,1,'Alex'), (3,'Laptop',1100,2,'Cairo'), (4,'Tablet',600,2,'Giza'), (5,'Phone',850,3,'Cairo'), (6,'Laptop',1300,3,'Alex'); """) # Query with pandas df = pd.read_sql_query("SELECT * FROM sales", conn) print(df)
Output
id product amount month region 0 1 Laptop 1200.0 1 Cairo 1 2 Phone 800.0 1 Alex ...

SQL Queries with pandas

python
# GROUP BY — total sales per product q1 = pd.read_sql_query(""" SELECT product, SUM(amount) as total, COUNT(*) as orders FROM sales GROUP BY product ORDER BY total DESC """, conn) print("Sales by product:\n", q1) # WHERE + aggregate q2 = pd.read_sql_query(""" SELECT region, AVG(amount) as avg_sale FROM sales WHERE month >= 2 GROUP BY region """, conn) print("\nAvg sale by region (from month 2):\n", q2) # Write DataFrame back to SQL df_new = pd.DataFrame({'product':['Camera'],'amount':[950],'month':[4],'region':['Giza'],'id':[7]}) df_new.to_sql('sales', conn, if_exists='append', index=False) print("\nRows after insert:", pd.read_sql_query("SELECT COUNT(*) FROM sales", conn).iloc[0,0])
🎯 TASKEmployee Database Query
Create an SQLite table 'employees' with columns: name, department, salary, hire_year. Insert 8 employees. Query: (1) average salary per department, (2) employees hired after 2020, (3) the top earner.
✅ Solution
python
import sqlite3, pandas as pd conn = sqlite3.connect(':memory:') conn.executescript(""" CREATE TABLE employees (name TEXT, department TEXT, salary REAL, hire_year INT); INSERT INTO employees VALUES ('Ahmed','Engineering',85000,2019),('Sara','Marketing',72000,2021), ('Mohamed','Engineering',92000,2018),('Nour','HR',65000,2022), ('Karim','Marketing',78000,2020),('Dina','Engineering',88000,2023), ('Omar','HR',60000,2019),('Layla','Marketing',82000,2021); """) print(pd.read_sql_query("SELECT department, AVG(salary) as avg FROM employees GROUP BY department", conn)) print(pd.read_sql_query("SELECT name, salary FROM employees WHERE hire_year > 2020", conn)) print(pd.read_sql_query("SELECT name, salary FROM employees ORDER BY salary DESC LIMIT 1", conn))
